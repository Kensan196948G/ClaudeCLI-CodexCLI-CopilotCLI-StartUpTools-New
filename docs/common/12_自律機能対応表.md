# 自律機能対応表

この表は、共通の自律開発思想を Claude Code / Codex CLI / GitHub Copilot CLI に適用したときの実現性を整理したものです。

凡例:
- `可能`: ツール標準機能としてかなり自然に実現できる
- `擬似実装`: 外側のスクリプトや運用で補う必要がある
- `不可`: その名前や機能をそのまま持ち込むのは不適切

| 機能 | Claude Code | Codex CLI | GitHub Copilot CLI |
|---|---|---|---|
| 自動モードループ | 可能 | 可能 | 可能 |
| Agent Teams 自動生成 | 可能 | 擬似実装 | 擬似実装 |
| SubAgent 自動割当 | 可能 | 擬似実装 | 可能 |
| 役割別ディスカッション可視化 | 可能 | 擬似実装 | 擬似実装 |
| 並列オーケストレーション | 可能 | 擬似実装 | 可能 |
| WorkTree 自動生成 | 擬似実装 | 擬似実装 | 擬似実装 |
| Hooks 並列実行 | 可能 | 擬似実装 | 可能 |
| CI 修復 AI | 可能 | 可能 | 可能 |
| Memory MCP / memory 保存 | 可能 | 擬似実装 | 擬似実装 |
| GitHub Issue / PR / CI 分析 | 可能 | 可能 | 可能 |
| 開発ダッシュボード | 擬似実装 | 擬似実装 | 擬似実装 |
| 自己進化プロンプト | 擬似実装 | 擬似実装 | 擬似実装 |
| Claude 固有の `claude-mem` | 可能 | 不可 | 不可 |

## ツール別の前提

### Claude Code

- 中心機能は Agent Teams
- 小さな分業は Subagents
- hooks / MCP / settings を組み合わせて自立性を上げる

### Codex CLI

- 中心機能は管理者-担当者オーケストレーション
- `exec`、`review`、`resume`、`fork` を軸に組み立てる
- Agent Teams は思想として取り込むが、実装は外側で補う

### GitHub Copilot CLI

- 中心機能はカスタムエージェント + fleet
- delegation、hooks、GitHub 連携を優先する
- Agent Teams は fleet と delegated execution に翻訳して扱う
