# 自律開発コア

この文書は、Claude Code / Codex CLI / GitHub Copilot CLI の 3 ツールに共通する自律開発思想を切り出したものです。

各ツール固有のテンプレート:
- Claude Code: [scripts/templates/CLAUDE.md](/D:/ClaudeCLI-CodexCLI-CopilotCLI-StartUpTools-New/scripts/templates/CLAUDE.md)
- Codex CLI: [scripts/templates/AGENTS.md](/D:/ClaudeCLI-CodexCLI-CopilotCLI-StartUpTools-New/scripts/templates/AGENTS.md)
- GitHub Copilot CLI: [scripts/templates/copilot-instructions.md](/D:/ClaudeCLI-CodexCLI-CopilotCLI-StartUpTools-New/scripts/templates/copilot-instructions.md)

## システム目的

3 ツールに共通する目的は次のとおりです。

- 自動設計
- 自動実装
- 自動検証
- 自動修復
- 開発知識蓄積

ただし実現方法はツールごとに異なります。

## 共通ループ

```text
状況分析
↓
役割分担の可視化
↓
タスク分解
↓
実装
↓
検証
↓
修復 or 完了
↓
次のアクション
```

## 共通ルール

- 可視化を優先する
- 役割分担を明示する
- 1 タスク = 1 成果物または 1 完了条件に寄せる
- 同一ファイルへの無秩序な並列編集を避ける
- `main` へ直接 push しない
- CI / test / lint / build を品質ゲートにする
- 無限修復ループを避ける

## 承認ルール

自動で進めてよいもの:
- 調査
- 設計
- 実装
- テスト
- レビュー
- 安全なコマンド実行

ユーザー確認を入れるもの:
- `push`
- `merge`
- `delete branch`
- `release`
- 破壊的変更
- 認証 / secret / 権限変更

## 出力順序

```text
1. 役割別ディスカッション
2. 設計決定
3. 実装
4. 検証
5. 次のアクション
```

## ツール別の翻訳

| 共通概念 | Claude Code | Codex CLI | GitHub Copilot CLI |
|---|---|---|---|
| 役割分担 | Agent Teams / Subagents | 管理者-担当者分担 | カスタムエージェント + fleet |
| 実行方式 | 対話 + agent team | `exec` / `review` 中心 | 委譲 / レビュー / 文書 / GitHub 操作 |
| 外部連携 | MCP / hooks | MCP / `config.toml` / オーケストレーター | MCP / hooks / GitHub 連携 |
| 記憶 | Memory MCP / ログ | ログ / MCP / 状態ファイル | instructions / ログ / MCP |

## 行動原則

```text
構造化思考
可視化
並列実行
継続改善
再現性
```

この 5 つは共通ですが、並列実行の方式はツールごとに異なります。
