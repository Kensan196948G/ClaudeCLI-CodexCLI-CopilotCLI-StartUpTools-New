# ツール比較ガイド

3つの AI CLI ツールの役割を、現行構成に合わせて整理します。

---

## 比較表

| 項目 | Claude Code | Codex CLI | GitHub Copilot CLI |
|------|-------------|-----------|--------------------|
| 提供元 | Anthropic | OpenAI | GitHub |
| 主用途 | 大規模変更、レビュー、実装支援 | コード生成、シェル支援 | `gh copilot` によるコマンド支援 |
| インストール | `npm install -g @anthropic-ai/claude-code` | `npm install -g @openai/codex` | `gh extension install github/gh-copilot` |
| 認証 | `/login` または API キー | `codex --login` または API キー | `gh auth login` |
| ローカル起動 | 対応 | 対応 | 対応 |
| SSH 起動 | 対応 | 対応 | 対応 |
| プロジェクト配備物 | `CLAUDE.md` | `AGENTS.md` | `.github/copilot-instructions.md` |

---

## 使い分け

### Claude Code を使う場面

- 複数ファイルの整合を保ちながら修正したい
- 実装だけでなくレビューや調査も任せたい
- 日本語中心で深い対話をしたい

### Codex CLI を使う場面

- すばやくコードやコマンドを生成したい
- ターミナル中心の開発を軽く回したい
- OpenAI 環境に寄せて運用したい

### GitHub Copilot CLI を使う場面

- `gh copilot suggest` / `gh copilot explain` を使いたい
- Git / GitHub 操作の補助がほしい
- GitHub CLI を日常的に使っている

---

## 最初に選ぶなら

- 実装とレビューをまとめて任せたい: Claude Code
- 軽快に CLI 支援を使いたい: Codex CLI
- GitHub 操作を補助したい: GitHub Copilot CLI

---

## このプロジェクト内での位置づけ

本リポジトリは、3ツールを競合させるのではなく、同じ設定体系と起動方式で共存させるためのランチャーです。用途に応じて `start.bat` または `Start-All.ps1` から切り替えてください。
