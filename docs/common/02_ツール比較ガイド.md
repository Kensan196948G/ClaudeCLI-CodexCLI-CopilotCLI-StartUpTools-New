## 廃止 (v3.1.0): Codex CLI / GitHub Copilot CLI

これらの起動メニュー (S2/S3/L2/L3) は v3.1.0 で削除されました。

| 項目 | Codex CLI (廃止) | GitHub Copilot CLI (廃止) |
|------|------------------|---------------------------|
| メニュー | S2 / L2 ❌ 削除 | S3 / L3 ❌ 削除 |
| 起動スクリプト | `Start-CodexCLI.ps1` (残置・無効化) | `Start-CopilotCLI.ps1` (残置・無効化) |
| config.json | `tools.codex.enabled = false` | `tools.copilot.enabled = false` |
| 復活方法 | `enabled = true` に戻して旧バージョンの Start-Menu.ps1 を参照 | 同左 |

> **誤解しやすい点**: Codex CLI (起動メニュー) は廃止されましたが、**Codex によるコードレビュー** (`/codex:review`, `/codex:rescue`) は ClaudeCode 内の Slash command として引き続き利用可能です。両者は別物です。

## 使い分け

### Claude Code を使うべき場面 (現行の唯一の選択肢)

- 複数ファイルの整合を保ちながら修正したい
- 実装だけでなくレビューや調査も任せたい
- 日本語中心で深い対話をしたい
- 週次で自動的に開発を進めたい (Cron)
- セッションの残り時間を Windows Terminal で可視化したい (Session Info タブ)
- ClaudeOS による自律開発を試したい

---

## 最初に選ぶなら

- 実装とレビューをまとめて任せたい: Claude Code
- 軽快に CLI 支援を使いたい: Codex CLI
- GitHub 操作を補助したい: GitHub Copilot CLI

---

## このプロジェクト内での位置づけ

本リポジトリは、3ツールを競合させるのではなく、同じ設定体系と起動方式で共存させるためのランチャーです。用途に応じて `start.bat` または `Start-All.ps1` から切り替えてください。
