# トラブルシューティング

現行構成でよくある問題と対処をまとめます。

---

## まず実行すること

```powershell
.\scripts\test\Test-AllTools.ps1
.\scripts\test\test-drive-mapping.ps1
```

`Test-AllTools.ps1` は現在、設定スキーマ、CLI コマンド、認証前提、共有パス、起動例をまとめて確認します。

MCP の startup / health / shutdown まで実環境で確認したい場合は、CI ではなく次を使います。

```powershell
pwsh -NoProfile -File .\scripts\test\Test-McpRuntime.ps1
pwsh -NoProfile -File .\scripts\test\Test-McpRuntime.ps1 -OutputFormat Json
```

## `claude` / `codex` / `gh` が見つからない

```powershell
claude --version
codex --version
gh --version
copilot --version
```

未導入なら:

```powershell
npm install -g @anthropic-ai/claude-code
npm install -g @openai/codex
gh extension install github/gh-copilot
```

## API キーまたはログインの問題

Claude Code:

```powershell
claude
# 起動後に /login
```

Codex CLI:

```powershell
codex --login
```

API 利用時:

```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-..."
$env:OPENAI_API_KEY = "sk-..."
```

Copilot:

```powershell
gh auth login
gh auth status
```

## `config.json` が見つからない

```powershell
Copy-Item config\config.json.template config\config.json
```

---

## SSH 接続できない

```powershell
ssh your-linux-host "echo OK"
ssh -vvv your-linux-host
```

確認項目:
- `linuxHost` が正しい
- SSH 鍵が配置されている
- Linux ホストに到達できる

---

## `sshProjectsDir` または共有ドライブにアクセスできない

```powershell
Test-Path Z:\
Test-Path "\\192.168.0.185\Projects"
.\scripts\test\test-drive-mapping.ps1
```

必要なら `config.json` の `sshProjectsDir` と `projectsDirUnc` を見直してください。

## PowerShell 実行ポリシー

```powershell
Get-ExecutionPolicy -List
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

一時的に実行する場合:

```powershell
powershell -ExecutionPolicy Bypass -File ".\scripts\main\Start-ClaudeCode.ps1"
```

```powershell
.\scripts\setup\setup-windows-terminal.ps1
```

## それでも解決しない場合

以下を添えて調査してください。

- Windows バージョン
- PowerShell バージョン
- 実行したスクリプト名
- エラーメッセージ全文
- `config/config.json` の機密情報を除いた内容
