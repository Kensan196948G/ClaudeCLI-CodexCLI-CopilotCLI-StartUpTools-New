# はじめに

このドキュメント群は、**Claude Code** を Windows から統一的に起動・運用するための現行ガイドです。

> **v3.1.0 で Claude Code 専用ツールに整理**: 旧バージョンでは Codex CLI / GitHub Copilot CLI の起動メニュー (S2/S3/L2/L3) も提供していましたが、v3.1.0 で削除されました。Claude Code 専用ランチャーとしての位置づけに集中し、Linux crontab 連携・Session Info タブ・Statusline グローバル適用に投資が集まっています。

---

## このプロジェクトでできること

- `start.bat` から Claude Code を選んで起動 (S1: SSH / L1: ローカル)
- Linux crontab で **週次自動起動** を登録 (メニュー 12)
- Windows Terminal に **Session Info タブ** を自動生成し、開始時刻 / 終了予定 / 残り時間をリアルタイム表示
- グローバル `~/.claude/settings.json` の `statusLine` を Linux 側へ **一括適用** (メニュー 13)
- ClaudeCode 内で `/cron-register` `/cron-cancel` `/work-time-set` 等の Slash command で対話的に制御
- 設定を `config/config.json` に集約
- 診断スクリプトで導入状態を確認

---

## 対応ツール

### Claude Code (唯一の起動対象)

- 複数ファイルをまたぐ実装・レビュー向き
- ローカル起動と SSH 起動の両方に対応
- ClaudeOS v8 (Agent Teams / Boot Sequence / Self Evolution / CodeRabbit 統合) と一体運用

### 廃止済み: Codex CLI / GitHub Copilot CLI

v3.1.0 で起動メニューから削除されました。`Start-CodexCLI.ps1` / `Start-CopilotCLI.ps1` ファイル自体は残置されていますが、`config.json` の `tools.codex.enabled = false` / `tools.copilot.enabled = false` で無効化されています。

> ※ **Codex (コードレビュー機能)** は別物として温存されています。`/codex:review` や `/codex:rescue` は ClaudeCode 内のコードレビュー支援として引き続き利用可能です。
