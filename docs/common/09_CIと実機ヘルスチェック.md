# CI と実機ヘルスチェック

このリポジトリでは、診断を `CI で常時回す範囲` と `実機でだけ回す範囲` に分けます。

## CI に入れる範囲

- `Invoke-Pester .\tests`
- `config/config.json.template` のスキーマ検証
- `Test-AllTools.ps1` の JSON スキーマ検証
- `TASKS.md` と Agent Teams 対応表の同期確認
- 旧構成参照の残存チェック

CI では、外部サービスや実ネットワークに依存しないことを優先します。

## 実機専用ヘルスチェック

- `.\scripts\test\Test-McpRuntime.ps1`
- `.\scripts\test\test-drive-mapping.ps1`
- SSH 実接続確認
- 実際の Claude / Codex / Copilot 認証状態の確認

実機専用チェックは、MCP サーバー起動、共有ドライブ、ネットワーク、資格情報に依存するため CI には入れません。

## Test-McpRuntime.ps1 の使い方

`Test-McpRuntime.ps1` は `Test-AllTools.ps1` の MCP runtime probe を有効化して実行します。

```powershell
pwsh -NoProfile -File .\scripts\test\Test-McpRuntime.ps1
pwsh -NoProfile -File .\scripts\test\Test-McpRuntime.ps1 -OutputFormat Json
```

明示的に設定ファイルを渡す場合:

```powershell
pwsh -NoProfile -File .\scripts\test\Test-McpRuntime.ps1 `
  -ConfigPath .\config\config.json `
  -McpConfigPath .\.mcp.json `
  -OutputFormat Json
```

補足:

- 現状は `pwsh` を推奨します。
- `powershell.exe` では、依存スクリプトの解析互換差異により実行に失敗する環境があります。

## 実機確認結果

2026年3月13日に `pwsh -NoProfile -File .\scripts\test\Test-McpRuntime.ps1 -OutputFormat Json` を実行した結果:

- `config/config.json` の読込は成功
- Claude / Codex / Copilot のコマンド検出は成功
- `.mcp.json` が存在しないため `mcp.configured = false`
- したがって、この時点では Memory MCP / 外部 MCP の実接続結果までは未確認

この結果は `MCP ランタイムが壊れている` のではなく、`実 MCP 設定がまだ無い` 状態を示します。
