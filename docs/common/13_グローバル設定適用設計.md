# グローバル設定適用設計

この文書は、Claude Code / Codex / GitHub Copilot CLI を起動する直前に、
各ツール向けのグローバル設定をどのファイルへどう反映するかを定義する。

前提:
- 実行モードは変更しない
- CLI オプションは変更しない
- 既存の `Start-*.ps1` の起動導線に追加する
- 反映対象は「安全にテンプレート化できる設定」に限定する

## 設計方針

グローバル設定反映は次の 3 層で行う。

1. 共通思想層
   - 自律開発コアの思想
   - 出力順序
   - 承認ルール
   - ループと停止条件
2. ツール固有テンプレート層
   - Claude: Agent Teams
   - Codex: 管理者-担当者オーケストレーション
   - Copilot: カスタムエージェント + fleet
3. ローカル環境層
   - API キー
   - MCP サーバー接続先
   - 認証状態
   - キャッシュやセッション

このうち、ランチャーが自動反映するのは 1 と 2 までとする。
3 はユーザー環境固有のため、テンプレート化せず環境変数や既存のログイン状態に任せる。

## 反映タイミング

全ツール共通:

1. `config/config.json` 読込
2. ツール有効確認
3. 実行モード確定
4. 対象プロジェクト確定
5. 実行前テンプレート同期
6. 環境変数適用
7. ツール本体起動

反映は必ず「対象プロジェクト確定後」「本体起動前」に行う。

## Claude Code

### 設計原本

- 主要テンプレート: [scripts/templates/CLAUDE.md](/D:/ClaudeCode-StartUpTools-New/scripts/templates/CLAUDE.md)
- 参照原本: [Sample/Claude/claudeos](/D:/ClaudeCode-StartUpTools-New/Sample/Claude/claudeos)

### 反映対象

| 種別 | 反映元 | 反映先 | 反映方法 |
|---|---|---|---|
| 全体方針 | `scripts/templates/CLAUDE.md` | `<project>/CLAUDE.md` | 毎回同期 |
| 運用知識 | `Sample/Claude/claudeos` を整理した正式版 | `<project>/.claude/claudeos/` | 初回作成 + 差分同期 |
| Claude 設定 | 今後追加する安全版テンプレート | `<project>/.claude/settings.json` | 上書きではなく統合 |
| MCP 設定 | 今後追加する安全版テンプレート | `<project>/.mcp.json` または `<project>/.claude/mcp.json` | 環境変数プレースホルダ維持 |
| Agent 定義 | 今後追加する役割テンプレート | `<project>/.claude/agents/*.md` | 差分同期 |

### 適用ルール

- `Sample/Claude/claudeos` はそのままコピーしない
- `sessions`、`cache`、`history`、`downloads`、認証情報は反映しない
- `claudeos/system`、`executive`、`management`、`loops`、`ci`、`agents`、`worktree` のみを正式テンプレート化して反映する
- `CLAUDE.md` は短い司令塔文書、詳細は `.claude/claudeos/` に分離する

### 実装メモ

将来的に `Sync-ProjectTemplate` の単体同期に加えて、ディレクトリ同期関数を追加する。

候補:
- `Sync-ProjectTemplateDirectory -TemplateDir ... -TargetDir ... -Filter ...`
- `Sync-LauncherClaudeGlobalConfig -ProjectDir ...`

### 反映対象

| 種別 | 反映元 | 反映先 | 反映方法 |
|---|---|---|---|
| 全体方針 | `Codex/AGENTS.md` | `<project>/AGENTS.md` | 毎回同期 |
| Codex 設定 | `scripts/templates/codex-config.toml` | `<project>/.codex/config.toml` | merge 優先 |
| MCP 設定 | `.codex/config.toml` 内の project-scoped 定義 | `<project>/.codex/config.toml` | テンプレート差し込み |
| 役割補助 | 今後追加する workflow docs | `<project>/.codex/workflows/*.md` | 初回作成 + 差分同期 |

### 適用ルール

- Claude の `Agent Teams` は Codex では「管理者-担当者オーケストレーション」に翻訳する
- `agents/` 相当の責務は `Manager / Architect / Build / Test / Review / Research / Ops` に再編する
- `Sample/Codex/auth.json` や state DB は反映しない
- `config.toml` は既存ユーザー設定を壊さないよう merge を前提にする
- `exec / review / resume / fork` を使う前提は文書へ反映し、オプションは変更しない

### 実装メモ

Codex は `AGENTS.md` と `.codex/config.toml` を同期する。ランチャー標準は `--full-auto` とし、`yolo` profile は限定用途で保持する。

候補:
- `Sync-LauncherCodexGlobalConfig -ProjectDir ...`
- `Merge-CodexProjectConfig -TemplatePath ... -TargetPath ...`

候補:
- `Sync-LauncherCopilotGlobalConfig -ProjectDir ...`
- `Sync-ProjectTemplateDirectory -TemplateDir CopilotCLI\\agents -TargetDir <project>\\.github\\agents`

## ローカル実行と SSH 実行

| モード | 方針 |
|---|---|
| Local | プロジェクトディレクトリへ直接同期してから起動する |
| SSH | 原則はリモートプロジェクト側にも同じテンプレートを置く必要がある |

SSH 実行時の設計方針:

- 第 1 段階: ローカル実行時のみグローバル設定反映を強化する
- 第 2 段階: SSH 実行時はリモート側へ template を転送して同期する
- 第 3 段階: ローカル / SSH 共通の同期関数に統一する

## 除外対象

次はテンプレートへ含めない。

- 認証トークン
- refresh token
- セッション履歴
- キャッシュ
- SQLite 状態ファイル
- ダウンロード履歴
- 端末固有パス
- trusted folders の実値

## 現時点の結論

既存ランチャーはすでに「単一ファイルの正式テンプレート同期」まではできている。
次段ではこれを「ツール別グローバル設定ディレクトリ同期」へ拡張するのが自然である。

その際の核は次の通り。

- Claude: `CLAUDE.md` + `.claude/claudeos/` + `.claude/settings.json` + `.mcp.json`
- Codex: `AGENTS.md` + `.codex/config.toml`
- Copilot: `.github/copilot-instructions.md` + custom agents + hooks + MCP
