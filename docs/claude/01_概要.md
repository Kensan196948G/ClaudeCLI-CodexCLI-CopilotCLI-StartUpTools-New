# Claude Code 概要

> 対象ツール: Claude Code（Anthropic）
> ドキュメント種別: 概要・特徴

---

## Claude Code とは

Claude Code は、Anthropic が提供するターミナルベースの AI コーディングアシスタントです。コード生成、調査、レビュー、リファクタリング、テスト支援を、プロジェクト全体の文脈を踏まえて進められます。

このリポジトリでは、Claude Code を単体で直接使うだけでなく、`Start-ClaudeCode.ps1` を通じてローカル起動と SSH 起動を切り替えられるようにしています。

---

## 主な特徴

### プロジェクト全体を前提に作業できる

- 複数ファイルにまたがる変更を進めやすい
- 実装、レビュー、テスト観点をまとめて扱える
- 日本語の指示でも運用しやすい

### ターミナル中心で使える

- CLI からそのまま起動できる
- プロジェクトディレクトリで直接作業できる
- ローカル環境でも SSH 先の Linux 環境でも使える

### プロジェクトごとの指示を持たせられる

- プロジェクト直下の `CLAUDE.md` を参照させられる
- このリポジトリの起動スクリプトは `scripts/templates/CLAUDE.md` を必要に応じて配備する

---

## このプロジェクトでの使い方

現行構成では、Claude Code の利用は次の流れです。

1. `start.bat` または `Start-ClaudeCode.ps1` を実行
2. ローカル起動か SSH 起動を選択
3. 対象プロジェクトを選択
4. 必要に応じて `CLAUDE.md` を配備
5. `tools.claude.args` と `tools.claude.env` を適用して起動

標準対象:
- ローカル: `projectsDir`
- SSH: `sshProjectsDir` と `linuxBase`

---

## 現行構成で自動化されるもの

- `config/config.json` の読み込み
- `claude` コマンドの存在確認
- プロジェクト選択
- `CLAUDE.md` テンプレート配備
- Claude 用環境変数の設定
- SSH 経由の Linux 上起動

現行の標準フローには、旧構成の DevTools ポート割り当て、tmux ダッシュボード、`run-claude.sh` 自動生成は含みません。

---

## 認証方法

Claude Code は次のどちらかで利用します。

- サブスクリプション認証
- `ANTHROPIC_API_KEY` による API 認証

サブスクリプション認証:

```powershell
claude
# 起動後に /login
```

API 認証:

```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-..."
```

## どんな場面に向くか
