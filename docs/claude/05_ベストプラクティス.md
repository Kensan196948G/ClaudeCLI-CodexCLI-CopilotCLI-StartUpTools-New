# Claude Code ベストプラクティス

> 対象ツール: Claude Code（Anthropic）
> ドキュメント種別: ベストプラクティス・活用パターン

---

## 基本方針

- 変更対象のファイルや制約を具体的に伝える
- 大きいタスクは段階に分ける
- 実装前に期待する結果を明示する
- `CLAUDE.md` でプロジェクト固有ルールを持たせる

---

## 良い指示の出し方

### 具体的に対象を示す

悪い例:

```text
バグを直してください
```

良い例:

```text
src/services/payment.service.ts の processPayment で、
amount <= 0 の場合に ValidationError を返すように修正してください。
関連テストは tests/payment.service.spec.ts にあります。
```

### 制約を先に伝える

```text
- Node.js 18 前提
- 外部ライブラリ追加なし
- TypeScript strict を維持
- 既存テストも更新する
```

### 期待する出力を伝える

```text
1. 変更方針
2. 修正ファイル
3. 実装
4. テスト観点
```

---

## `CLAUDE.md` を活用する

プロジェクトルートに `CLAUDE.md` を置くと、毎回説明する情報を減らせます。

例:

```markdown
# CLAUDE.md

## アーキテクチャ
- React + TypeScript
- Fastify
- Prisma

## 規約
- 変数名は camelCase
- console.log は使わない
- テストは Vitest
```

このリポジトリでは `Start-ClaudeCode.ps1` がテンプレートを必要に応じて配備します。

---

## コンテキスト管理

- 長い作業では `/status` で状態を確認する
- 区切りのよいタイミングで `/compact` を使う
- 重要な前提は会話の早い段階で伝える

長期作業では、作業メモをファイルに残すとセッションをまたいで安定します。

---

## 起動方法の使い分け

### ローカル起動が向くケース

- Windows 側のローカルプロジェクトをすぐ触りたい
- SSH 接続なしで軽く作業したい

```powershell
.\scripts\main\Start-ClaudeCode.ps1 -Project "my-project" -Local
```

### SSH 起動が向くケース

- Linux 側の開発環境を使いたい
- リモート環境でそのままビルドやテストを回したい

```powershell
.\scripts\main\Start-ClaudeCode.ps1 -Project "my-project"
```

---

## セキュリティ注意

### `--dangerously-skip-permissions`

このリポジトリの設定では、Claude Code に `--dangerously-skip-permissions` を渡せます。利便性は高いですが、開発環境専用と考えてください。

使う前提:
- 信頼できるプロジェクト
- Git で差分管理できる
- 本番環境ではない

### API キー管理

- API キーをソースに書かない
- `config.json` に平文で固定しない
- ユーザー環境変数かログイン認証を使う

## 実務での進め方

### 小さく区切る

大きなタスクは次のように分けると安定します。

```text
1. 影響範囲を調査
2. 実装
3. テスト更新
4. 差分確認
```

### 作業前後に Git を見る

```powershell
git status
git diff
```

### 非対話や確認用のオプションを使う

```powershell
.\scripts\main\Start-ClaudeCode.ps1 -Project "my-project" -DryRun
.\scripts\main\Start-ClaudeCode.ps1 -Project "my-project" -NonInteractive
```

---

## よくある失敗

- 指示が曖昧で、修正範囲が広がる
- 大きい変更を一度に頼みすぎる
- 作業前に制約を伝えていない
- 差分確認をせずに次へ進む

---

## 現行仕様で前提にしないもの

次は現行の標準運用ではありません。

- DevTools 連携前提の作業
- tmux ダッシュボード前提の運用
- `run-claude.sh` 生成前提の説明
- statusline 再読込前提の運用

必要なら個別に追加運用できますが、標準仕様は `Start-ClaudeCode.ps1` ベースです。
