# GitHub Copilot CLI セットアップ

## 1. GitHub CLI を準備する

Copilot CLI は `gh copilot` を前提にしています。まず `gh` コマンドが利用可能であることを確認します。

```powershell
gh --version
gh auth status
```

未ログインの場合は認証します。

```powershell
gh auth login
```

## 2. 設定ファイルを用意する

```powershell
Copy-Item .\config\config.json.template .\config\config.json
```

`config/config.json` では次を確認します。

- `tools.copilot.enabled`
- `tools.copilot.command`
- `tools.copilot.args`
- `tools.copilot.env`

標準では次のような構成を使います。

```json
{
  "tools": {
    "copilot": {
      "enabled": true,
      "command": "gh",
      "args": ["copilot"],
      "env": {}
    }
  }
}
```

## 3. 起動確認

ローカル実行の確認:

```powershell
.\scripts\main\Start-CopilotCLI.ps1 -Project <プロジェクト名> -Local -DryRun
```

SSH リモート実行の確認:

```powershell
.\scripts\main\Start-CopilotCLI.ps1 -Project <プロジェクト名> -DryRun
```

## 4. 配備されるファイル

Copilot 起動時には、必要に応じて `.github/copilot-instructions.md` がプロジェクト側へ配備されます。
