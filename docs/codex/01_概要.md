# Codex CLI 概要

このリポジトリでは、Codex CLI を Windows から統一的に起動するために [scripts/main/Start-CodexCLI.ps1](/D:/ClaudeCLI-CodexCLI-CopilotCLI-StartUpTools-New/scripts/main/Start-CodexCLI.ps1) を使用します。

Codex CLI の運用方針は次のとおりです。

- ローカル実行と SSH リモート実行の両方に対応する
- 設定は [config/config.json.template](/D:/ClaudeCLI-CodexCLI-CopilotCLI-StartUpTools-New/config/config.json.template) を基準にし、実環境は `config/config.json` で上書きする
- プロジェクト起動時に [scripts/templates/AGENTS.md](/D:/ClaudeCLI-CodexCLI-CopilotCLI-StartUpTools-New/scripts/templates/AGENTS.md) を配備し、Codex 向けの manager-worker orchestration を揃える
- 直接コマンドを叩くよりも、起動スクリプト経由の運用を標準とする

主な実行モードは 2 つです。

- ローカル実行: Windows 側の作業ディレクトリで `codex` を起動する
- SSH リモート実行: Linux 側の作業ディレクトリで `codex` を起動する

共通思想は [11_自律開発コア.md](/D:/ClaudeCLI-CodexCLI-CopilotCLI-StartUpTools-New/docs/common/11_自律開発コア.md) を参照してください。Codex 固有の翻訳は manager-worker orchestration です。
