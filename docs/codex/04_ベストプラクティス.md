# Codex CLI ベストプラクティス

- Codex CLI は直接起動ではなく [scripts/main/Start-CodexCLI.ps1](/D:/ClaudeCLI-CodexCLI-CopilotCLI-StartUpTools-New/scripts/main/Start-CodexCLI.ps1) 経由を標準にする
- `config/config.json.template` を基準に管理し、環境依存値は `config/config.json` に限定する
- プロジェクトごとの運用ルールは `AGENTS.md` に集約し、手動コピーではなくテンプレート配備に任せる
- 自動化やバッチ実行では `-NonInteractive` と `-DryRun` を使い分ける
- SSH リモート実行を使う場合は、Windows 側と Linux 側で同じプロジェクト名が解決できるようにディレクトリ構成を揃える
