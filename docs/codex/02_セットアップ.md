# Codex CLI セットアップ

## 1. Codex CLI を利用可能にする

まず `codex` コマンドが Windows 側または Linux 側で実行できる状態にします。

PowerShell では次で確認できます。

```powershell
codex --version
```

認証が必要な環境では、先にログインを済ませます。

```powershell
codex --login
```

## 2. 設定ファイルを用意する

初回セットアップではテンプレートをコピーして実ファイルを作成します。

```powershell
Copy-Item .\config\config.json.template .\config\config.json
```

`config/config.json` では最低限次を確認します。

- `projectsDir`
- `projectsDirUnc`
- `linuxHost`
- `linuxBase`
- `sshProjectsDir`
- `tools.codex.enabled`
- `tools.codex.command`
- `tools.codex.args`
- `tools.codex.env`

## 3. 起動確認

ローカルでの疎通確認:

```powershell
.\scripts\main\Start-CodexCLI.ps1 -Project <プロジェクト名> -Local -DryRun
```

SSH リモート実行の疎通確認:

```powershell
.\scripts\main\Start-CodexCLI.ps1 -Project <プロジェクト名> -DryRun
```

`-DryRun` では実行コマンドだけを確認できます。

## 4. 配備されるファイル

Codex CLI 起動時には、必要に応じてプロジェクト直下に [scripts/templates/AGENTS.md](/D:/ClaudeCLI-CodexCLI-CopilotCLI-StartUpTools-New/scripts/templates/AGENTS.md) が配備されます。

これは Codex に対する運用ルールの基準ファイルです。手で毎回作成する前提ではありません。
