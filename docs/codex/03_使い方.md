# Codex CLI 使い方

## 基本起動

推奨する起動方法は [scripts/main/Start-CodexCLI.ps1](/D:/ClaudeCode-StartUpTools-New/scripts/main/Start-CodexCLI.ps1) を使う方法です。

### ローカル実行

```powershell
.\scripts\main\Start-CodexCLI.ps1 -Project MyProject -Local
```

### SSH リモート実行

```powershell
.\scripts\main\Start-CodexCLI.ps1 -Project MyProject
```

### 非対話モード

```powershell
.\scripts\main\Start-CodexCLI.ps1 -Project MyProject -Local -NonInteractive
```

### 実行確認のみ

```powershell
.\scripts\main\Start-CodexCLI.ps1 -Project MyProject -Local -DryRun
```

## 起動時の動作

起動スクリプトは次の処理を行います。

1. `config/config.json` を読み込む
2. Codex CLI が利用可能か確認する
3. プロジェクトディレクトリを決定する
4. 必要なら `AGENTS.md` を配備する
5. `tools.codex.env` の環境変数を設定する
6. `codex` コマンドを実行する

## よく使うオプション

- `-Project`: 対象プロジェクト名を指定する
- `-Local`: Windows 側で実行する
- `-NonInteractive`: 確認プロンプトを出さずに進める
- `-DryRun`: 実際には起動せず、実行予定のコマンドだけを表示する

## 直接起動との違い

`codex` を直接叩くこともできますが、起動スクリプト経由には次の利点があります。

- 設定ファイルから作業ディレクトリを統一できる
- `AGENTS.md` を自動配備できる
- ローカルと SSH の切り替えを同じ引数体系で扱える
- 将来の共通処理追加をスクリプト側に集約できる
